四大产品支柱

清晰、精准、可执行、不断线。

所有能力都服务于同一件事：让 Agent 生成的 Markdown 更容易理解、审阅和清晰交回。

01

AI 智能提炼OpenAI 或 DeepSeek

将冗长的 Markdown 提炼为一个中心结论和 3–5 个关键要点，每条 AI 洞察都附带原文证据。

用户使用自己的 API Key，密钥保存在 macOS Keychain。只有主动触发 AI 功能时，Sigla 才会连接所选服务商。

02

轻松框选批注圈出原文，写下意见

圈出任意段落，即可添加评论、问题或修改要求；每条批注都精准绑定原文。

批注保存在 Sigla 私有数据库。外部修改会触发锚点核对，匹配不确定时由用户确认。

03

Agent 可执行反馈从审阅直接进入修改

把选中的批注整理成清晰任务，包含准确位置、具体要求和验收标准。

反馈可直接交回 Codex 或其他 Agent。相关与冲突要求保持清晰，用户原话和原文证据完整保留。

04

离线连续性审阅无需依赖联网

无需 API Key 或网络连接，也能阅读、渲染、批注和生成本地基础反馈。

联网 AI 只在用户选择后运行。源 Markdown 始终只读，所有审阅状态保存在本机私有空间。

常见问题

下载之前，先了解清晰边界。

Sigla 开源吗？

当前没有开源。公开仓库包含产品介绍、社区资料和安装包，应用源代码采用专有软件许可并保持私有。

Sigla 会修改 Markdown 吗？

Sigla 不提供源 Markdown 的修改、删除、移动、重命名、复制、恢复或格式化操作。批注和审阅状态独立保存。

没有 AI 可以使用吗？

可以。没有 API Key 或网络连接时，阅读、渲染、批注、文件变化监听和本地基础反馈仍然可用。

支持哪些 AI 服务？

Sigla 1.0 预设支持 OpenAI 和 DeepSeek，采用用户自备 API Key 的方式。AI 功能可选并由用户主动触发。

Sigla 会上传我的文档吗？

Sigla 没有文档同步服务器。用户主动执行 AI 功能时，必要的文档内容和审阅上下文会直接发送至所选服务商。
