## Capture rules

- Use only fixtures or purpose-written demonstration Markdown.
- Remove account names, absolute paths, API keys, private projects, and notification content.
- Capture the installed release candidate at a consistent window size.
- Export PNG at 2× density; keep text readable after GitHub scales the image.
- Use the final Sigla light/dark themes without decorative mock-device frames.
- Confirm every visible capability exists in the shipped build.
